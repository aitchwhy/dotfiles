# Task ID: 2
# Title: Implement POST /templates/runs Endpoint
# Status: pending
# Dependencies: 1
# Priority: high
# Description: Create the REST endpoint to trigger a new template-population Run, optionally within an existing Stem, as specified in FLO-235.
# Details:
1. Create `postRun.ts` controller in `gateways/noggin/src/chats/controllers/`
2. Implement validation using Zod for the request body:
```typescript
const postRunSchema = z.object({
  template_ref: z.string(),
  clinicals_ref: z.array(z.string()).min(1),
  stem_id: z.string().uuid().optional()
});
```
3. Implement controller logic to:
   - Validate template existence
   - Create a new Run record in the database
   - If stem_id is provided, validate it exists
   - If stem_id is not provided, create a new Stem record
   - Return 201 with the stem_id
4. Update `routes.ts` to mount the endpoint:
```typescript
router.post('/templates/runs', postRunController);
```
5. Implement error handling for 404 (template not found) and validation errors

# Test Strategy:
1. Unit tests for controller with mocked dependencies
2. Test validation logic with valid and invalid payloads
3. Test error handling for non-existent templates
4. Integration tests with in-memory database
5. Test both scenarios: with and without stem_id provided
