# Architectural Decisions

This directory stores ADR (Architecture Decision Records) extracted from sessions.

## Format

Each decision is stored as `ADR-NNN-{slug}.md`:

```markdown
# ADR-001: Use Effect-TS for Error Handling

## Status
Accepted | Superseded | Deprecated

## Context
What is the issue that we're seeing that motivates this decision?

## Decision
What is the change that we're proposing and/or doing?

## Consequences
What becomes easier or more difficult because of this change?
```

## Creating Decisions

When making significant architectural choices:

1. Document the decision as an ADR
2. Save to this directory with next available number
3. Reference from relevant code/docs
